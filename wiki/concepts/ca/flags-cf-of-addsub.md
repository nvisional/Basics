# 标志位与加减运算 — CF、OF 与统一加减 ALU

> 一句话母题：**一串二进制位本身不知道自己是有符号还是无符号**，
> 所以硬件必须给出两个不同的"溢出灯"——**CF 管无符号，OF 管有符号**。
> 二刷别把这两盏灯（和它们各自的异或门）混为一谈。

---

## 直觉：结果位串是"沉默"的

减法在 CPU 里其实是补码加法：`A - B = A + (~B + 1)`。
加法器吐出的只是一串位，它**不携带"我该被当成有/无符号"的信息**：

- 当成**无符号**：最高位只是普通数据位，"减出负数"只能靠**借位**判断 → 看 **CF**
- 当成**有符号**：最高位是符号位，溢出 = 符号位被冲掉 → 看 **OF**

同一次运算，两盏灯可以给出**不同结论**，因为它们问的是两个问题。

---

## CF（进位 / 借位标志）— 给无符号用

只跟**最高位的一根进位线** `C_out` 有关，**不涉及内部进位的异或**：

```
CF = C_out ⊕ Sub        (Sub: 0=加法, 1=减法)
```

- 加法 `Sub=0`：`CF = C_out`，就是真·进位
- 减法 `Sub=1`：`CF = ~C_out`，即"借位"（x86 borrow 约定）

"减法时 CF = 进位取反"和"CF = C_out ⊕ Sub"是**同一件事**：取反 = `⊕1`，而减法时 `Sub` 恰好是 1。

---

## OF（溢出标志）— 给有符号用，这才是"内部进位异或门"

```
OF = C_out(最高位) ⊕ C_in(最高位)
   = 最高位进位 ⊕ 次高位进位
```

直觉：进入符号位的进位和从符号位出去的进位**不一致**，说明符号位被数值溢出污染了。
⚠️ 常见记串点：异或的是**次高位进位**，不是"最低位进位"；而且这是 **OF** 的判据，**不是 CF**。

---

## 统一加减 ALU：一根 Sub 线干三件事

加减共用一个加法器，靠 `Sub` 控制线复用：

```
结果 = A + (B ⊕ Sub) + Sub
            └─────┘    └─ 作为最低位进位 C_in
         Sub=1 时按位取反 B，再 +1 → 正好凑出 -B 的补码
```

同一根 `Sub` 还接到进位标志输出端做异或。于是 `Sub=1` 时它同时：
**① 取反 B ② 补最低位的 +1 ③ 翻转 CF**——这就是加减能共用一套电路的关键。

---

## 例子：4 位 `3 - 5`（Sub=1）

`3 + (~5+1) = 0011 + 1011`，把进位链标出来：

```
       bit3 bit2 bit1 bit0
进位链:  0    0    1    1
  0011
+ 1011
------
  1110          结果 = 1110
```

- C_in(最高位) = bit2 产生的进位 = **0**
- C_out(最高位) = bit3 产生的进位 = **0**

| 标志 | 公式 | 值 | 含义 |
|------|------|----|------|
| **CF** | `C_out ⊕ Sub = 0 ⊕ 1` | **1** | 无符号借位 → 3<5，无符号减出负（1110=14 非正确无符号答案）|
| **OF** | `C_out ⊕ C_in = 0 ⊕ 0` | **0** | 有符号没溢出（-2 在 4 位 -8~7 内，合法）|

反例 `5 - 3`：`0101+1101=1_0010`，`C_out=1` → `CF = 1⊕1 = 0`（无借位），结果 `0010=2` 正确。

---

## 两个异或门的对照（核心记忆）

| 标志 | 公式 | 异或的是 | 服务对象 |
|------|------|---------|---------|
| **CF** | `C_out ⊕ Sub` | 最高位进位 ⊕ **操作控制位** | 无符号 |
| **OF** | `C_out ⊕ C_in` | 最高位进位 ⊕ **次高位进位** | 有符号 |

一句话：CF 异或"控制线"，OF 异或"内部进位"。
