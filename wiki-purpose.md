# Wiki Purpose — 操作系统学习 Wiki 目标定义

## 为什么建这个 Wiki

系统性地学习操作系统，从底层机制（内存管理、进程调度、文件系统、I/O）出发，
逐步构建完整的 OS 知识图谱。每个概念消化后固化为一个知识原子，**知识只编译一次，持续积累**。

## 核心目标

1. **理解"为什么"** — 不只是知道机制怎么工作，更要理解设计决策背后的权衡
2. **自底向上** — 从硬件接口（MMU、中断、寄存器）出发，逐层向上到内核抽象
3. **可追溯** — 每个概念都关联来源，知道"这个理解是从哪里来的"
4. **可复习** — 结构化、可搜索、可交叉引用，随时捡起来

## 学习范围

| 模块 | 覆盖内容 | 优先级 |
|------|---------|--------|
| Memory Management | 分页、分段、虚拟内存、TLB、页面置换、伙伴系统 | 🔴 进行中 |
| Process & Concurrency | 进程模型、线程、锁、信号量、死锁、调度算法 | 🟡 待开始 |
| File System | inode、VFS、日志、ext4、NFS | 🟡 待开始 |
| I/O & Device | 中断、DMA、块设备、字符设备、驱动模型 | ⚪ 远期 |
| Networking | socket、TCP/IP 栈、零拷贝 | ⚪ 远期 |
| Virtualization | CPU 虚拟化、内存虚拟化、hypervisor | ⚪ 远期 |

## 学习方式

- **教材 + 源码阅读**作为 Raw Sources
- 讨论消化后 LLM 整理为概念节点
- 定期 lint 检查知识一致性
- 每完成一个模块，写 synthesis 做全局总结
