# Wiki Purpose — 计算机基础学习 Wiki 目标定义

## 为什么建这个 Wiki

系统性地学习计算机科学四大基础课，从底层硬件到上层网络，构建完整的 CS 基础知识图谱。
每个概念消化后固化为一个知识原子，**知识只编译一次，持续积累**。

## 核心目标

1. **理解"为什么"** — 不只知道机制怎么工作，更要理解设计决策背后的权衡
2. **自底向上** — 从硬件（晶体管、Cache、MMU）到抽象（进程、协议、算法）
3. **跨学科关联** — 一个概念可能在多个学科出现（如"缓存"在组成原理、OS、网络中都有体现）
4. **可追溯** — 每个概念关联来源，知道"这个理解是从哪里来的"
5. **可复习** — 结构化、可搜索、交叉引用，随时捡起来

## 学习范围与优先级

### 操作系统
| 模块 | 覆盖内容 | 进度 |
|------|---------|------|
| Memory Management | 分页、分段、虚拟内存、TLB、页面置换、伙伴系统 | 🔴 进行中 |
| Process & Concurrency | 进程模型、线程、锁、信号量、死锁、调度算法 | 🟢 已开始 |
| File System | inode、VFS、日志、ext4 | 🟡 待开始 |
| I/O & Device | 中断、DMA、块设备、驱动模型 | ⚪ 远期 |

### 计算机组成原理
| 模块 | 覆盖内容 | 进度 |
|------|---------|------|
| Data Representation | 整数/浮点数编码、ALU | ⚪ 远期 |
| Instruction Set | RISC-V / x86、寻址方式、调用约定 | ⚪ 远期 |
| Processor | 单周期、流水线、分支预测、乱序执行 | ⚪ 远期 |
| Memory Hierarchy | Cache 映射/替换/写策略、虚拟内存联动 | ⚪ 远期 |

### 计算机网络
| 模块 | 覆盖内容 | 进度 |
|------|---------|------|
| Physical & Link | 编码、帧、MAC、交换机 | ⚪ 远期 |
| Network | IP、路由、子网划分、NAT | ⚪ 远期 |
| Transport | TCP/UDP、拥塞控制、可靠传输 | ⚪ 远期 |
| Application | HTTP、DNS、TLS | ⚪ 远期 |

### 数据结构
| 模块 | 覆盖内容 | 进度 |
|------|---------|------|
| Linear | 数组、链表、栈、队列 | ⚪ 远期 |
| Tree & Graph | BST、AVL、B+Tree、最短路径、MST | ⚪ 远期 |
| Hash & Index | 哈希表、Bloom Filter、倒排索引 | ⚪ 远期 |
| Advanced | 跳表、并查集、线段树 | ⚪ 远期 |

## 学习方式

- **教材 + 源码 + 课程视频** 作为 Raw Sources
- 讨论消化后 LLM 整理为概念节点
- 定期 lint 检查知识一致性
- 跨学科概念（如"缓存"）用 `[[wikilinks]]` 连接不同学科的视角
- 每完成一个模块，写 synthesis 做总结
